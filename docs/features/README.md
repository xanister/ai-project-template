# Feature Issue Template

When filing a feature request, create a GitHub Issue with the title format `FEAT: short description` and the body below.

**Labels**: `type:feat` + one of `priority:high`, `priority:medium`, `priority:low`

---

## Description

What the feature is and why it's needed. Who benefits from it?

## Plan

**Done when:**
- [ ] [Specific, testable outcome 1]
- [ ] [Specific, testable outcome 2]

**Files to change:**
- `path/to/file.ts` — [what changes and why]
- `path/to/other.ts` — [what changes and why]

**Tests:**
- [Test case 1: what scenario, what's expected]
- [Test case 2: what scenario, what's expected]

---

## For Large Features (multi-phase or multi-day work)

Include the following additional sections:

**Phases:**
1. Phase 1 — [description]
2. Phase 2 — [description]

**Risks/dependencies:**
- [Any blockers, unknowns, or external dependencies]
